# Contributing

Thanks for your interest in contributing to WorkoutDB.

## Development setup

- **Client**: see `client/README.md`
- **Server**: see `server/.env.example` and the root `README.md`

## Branches and PRs

- Keep PRs focused and small when possible.
- Include a short description of the change and why it’s needed.
- If you change API behavior, include the endpoint(s) affected and an example request/response.

## Code style

- Prefer clear names and small functions.
- Avoid committing secrets (`.env`, credentials, tokens).
- Don’t commit `node_modules/` or build output.

## Reporting issues

If something breaks, include:

- Steps to reproduce
- Expected vs actual behavior
- Screenshots (for UI issues)
- Environment (OS, browser, Node version)
